Spikey Software¶

Preparation of operating system¶

We consider the newest stable Ubuntu operating system as reference system.

Software packages required for build:

sudo apt-get install libboost-all-dev libusb-1.0-0-dev liblog4cxx10-dev libqt4-dev libgtest-dev libgsl0-dev python-nose

Software packages required for experiment execution:

sudo apt-get install python-numpy python-scipy python-matplotlib

Installing hardware drivers¶

The driver software for the Spikey system is managed by the repository called “symap2ic”. Note that you need an account for the project management system in Heidelberg to access this repository. To get access, download and install the driver software process the following instructions step by step:

  1. Add ssh key for authentication (for help see here)
  2. Download the root repository: git clone git@gitviz.kip.uni-heidelberg.de:symap2ic.git
  3. cd symap2ic
  4. Set environment variables: . bootstrap.sh.UHEI
  5. Choose your software configuration: ./waf set_config stage1-flyspi-single
  6. Install the software: ./waf configure build install
  7. If using multiple chip connected to single host computer, create a file named “my_stage1_station” in your home directory and enter your default station (e.g., “station500”)

Installing PyNN¶

The Spikey backend is not included in the official PyNN project. So please download and install a Spikey-specific PyNN version which builds on PyNN version 0.6.0.

  • On UHEI computer
module load pynn/0.6.0-hw
  • On private computer
  1. Download the source code: git clone git@gitviz.kip.uni-heidelberg.de:deb-pynn
  2. cd deb-pynn
  3. Select branch: git checkout 0.6.0
  4. Install PyNN to the installation path DIR: python setup.py install --prefix=DIR
  5. Add the following line to your bashrc: export PYTHONPATH=$PYTHONPATH:DIR/lib/pythonX.Y/site-package (replace DIR, X, Y, respectively)

Allow access to USB device¶

If using the Spikey system on a private computer, you will need root privileges to allow normal users to access this USB device.

  1. Add your user to plugdev group: sudo usermod -aG plugdev USERNAME (replace USERNAME)
  2. Assign Spikey systems to plugdev group: cd symap2ic; sudo sh components/vmodule/nosudo
